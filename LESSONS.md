# LESSONS

## 2026-07-09 — 확률 시뮬레이션 비율 계산은 분모에 노이즈를 주면 100%를 넘을 수 있다
`runPolicy()`의 해소율 계산에서 `absorbed = Math.min(CARE_SLOT*fillRate, localUnmet*(1+gauss(0,0.08)))` 형태로 분모(localUnmet) 쪽에도 노이즈를 줬는데, 이 노이즈가 양의 방향으로 튀면 `absorbed`가 실제 `localUnmet`(노이즈 없는 원래 값)보다 커져서 `absorbed/localUnmet > 1`이 나옴. Monte Carlo 24개 시민모드 조합 테스트에선 안 걸렸고(그쪽은 이미 `Math.min(1, wait)`로 clamp돼 있었음), 정책모드 10회 반복 테스트에서 우연히 향남·우정처럼 시설정원(60명)이 미충족 인원(28~52명)보다 큰 케이스에서 90%CI 상단이 109~110%로 나와 발견.
교훈: 비율(ratio)을 몬테카를로로 뽑을 때, 분자·분모 둘 다에 독립 노이즈를 주면 논리적 상한(100%)을 넘을 수 있다는 걸 항상 의심할 것. 출력 직전에 `Math.min(1, ratio)` 같은 논리적 경계 clamp를 명시적으로 걸어야 함 — "몬테카를로니까 알아서 수렴하겠지"는 틀린 가정.

## 2026-07-09 — "완벽하다"는 보고와 실제 검증은 다르다
한 번 눌러본 결과가 잘 나왔다고 "다 됐다"고 보고했다가, 사용자가 "임의의 테스트 결과가 이미 정해진 데이터셋에 원하는 결과 화면이 아니라, 실제 서비스처럼 돌아가야 한다"고 재확인을 요구. 이후 24개 입력조합×3회(72회) + 정책모드 10회 반복을 헤드리스 브라우저로 실제 실행해서야 위 100%초과 버그를 잡음.
교훈: UI 검증은 "한 번 클릭해서 스크린샷 확인" 수준이 아니라 입력 공간 전체 + 확률적 변동성(반복 실행)까지 커버해야 진짜 검증이다. 특히 몬테카를로/랜덤 요소가 있는 코드는 1회 실행 무결점이 전체 무결점을 보장하지 않는다.
